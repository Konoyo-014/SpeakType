# v2.1.3 Readiness And Acceleration Design

## Goal

SpeakType v2.1.3 should make the local runtime feel ready, explainable, and faster without changing the product contract. Audio, raw transcription, and polished text remain local. ASR stays fixed on Qwen3-ASR via mlx-audio. Whisper Mode remains adaptive and automatic. Ollama remains the local LLM path for polish and translation, not a cloud service.

The version has two connected themes. The first theme is local readiness: the app should show whether microphone access, macOS input permissions, ASR model cache, Ollama service state, Ollama model availability, and the current target input field are usable. The second theme is latency: after v2.1.1 moved expensive work earlier, v2.1.3 should remove avoidable fixed waits and overlap safe warmup tasks.

## Readiness Experience

The new user-facing surface is a local diagnostics window opened from the menubar. It does not send any data anywhere and does not inspect dictated text. It only reads local state and reports concise results. The user should be able to see whether SpeakType can record, transcribe, polish, and insert before trying another dictation.

The readiness checks should be deliberately practical. Permission checks use the existing permission reader. Microphone checks verify that an input device can be discovered. ASR checks report whether the configured Qwen model is cached and whether the current engine is already loaded. Ollama checks distinguish the binary being installed from the HTTP service running on `localhost:11434`, then check whether the configured model exists. Target input checks inspect the current frontmost app and focused Accessibility element without inserting any text.

The window should give direct repair language rather than raw internals. A failed Ollama service check should say to open Ollama.app, run `brew services start ollama`, or temporarily run `ollama serve`. A missing model check should show the exact `ollama pull` command. Missing input permissions should tell the user to grant Accessibility and Input Monitoring, then restart SpeakType if permissions were just changed. A target input warning should tell the user to click an editable field and retry.

## Acceleration Strategy

The first speed change is startup overlap. In v2.1.2, setup loads ASR and only checks or prewarms Ollama after ASR loading completes. In v2.1.3, ASR loading and local LLM readiness/prewarm can run in parallel because they do not share mutable transcript data and both stay local. This improves first-use readiness when Qwen loading and Ollama cold-start would otherwise stack.

The second speed change is ASR inference warmup. Loading the Qwen model is not the same as warming the MLX/Metal inference path. After the model loads, SpeakType can run one tiny local dummy inference in the background to trigger kernel setup before the user's first real dictation. This must use the existing inference lock in non-blocking mode so a real recording or preview pass always wins.

The third speed change is local snapshot loading for cached Qwen models. If HuggingFace already has the configured Qwen3-ASR snapshot on disk, SpeakType should pass that local snapshot path to mlx-audio first. If mlx-audio rejects the local path, it falls back to the previous model-id path. This avoids turning every cached startup into an avoidable remote-resolution-shaped load path while preserving the old behavior as a recovery path.

The fourth speed change was evaluated and then withdrawn from the default final path. Conservative local VAD trimming looked attractive for latency, but short Mandarin phrases lost enough acoustic context to push Qwen toward near-homophones. The final Qwen transcription should therefore preserve the full captured audio buffer by default. Edge measurements can still be logged for diagnostics, but they should not change the inserted transcript path unless a later quality-gated design proves safe.

The fifth speed change is adaptive streaming-preview scheduling. Chunked preview already performs local Qwen transcription during the recording, but it is not a quality-equivalent replacement for the final Qwen pass. In testing, using preview-derived text as final output produced duplicate fragments and worse Chinese transcription. Preview therefore remains display-only, while quiet-tail scheduling is still useful because it keeps the UI responsive without changing the inserted transcript.

The sixth speed change is final-path preservation. Short-tail ASR merge was evaluated and then disabled because it can create duplicated or mixed-script fragments when the preview and tail pass disagree. The final inserted text should come from the normal Qwen final transcription path, with safe acceleration coming from local cache loading, warmup, trimming, and insertion/LLM latency reductions instead.

The seventh speed change is Ollama chat-path warmup while recording. Startup can keep the model resident, but real polish uses `/api/chat`, so recording start should send a tiny one-token local chat warmup when polish or translation is enabled. This uses no dictated text and should not run alongside a separate generate prewarm on the same recording.

The eighth speed change is bounded local LLM generation budgets. Short dictations should not give Ollama hundreds or thousands of unnecessary output tokens. Polish, translation, and edit requests can use conservative bounded budgets that still leave enough room for natural punctuation and translation expansion but reduce tail latency on ordinary short utterances.

The ninth speed change is target-aware insertion routing. Codex, Claude, Chrome, ChatGPT, Gemini, and similar Electron or browser-hosted editors often report Accessibility success without mutating `AXValue`, then accept the later paste. For these targets, the fastest reliable path is paste-first. Native text controls can still try Accessibility first because that remains the cleanest and fastest path when it is actually supported.

The tenth speed change is repeated target preparation caching. `AXManualAccessibility` only needs to be set once for a running process. Reusing that preparation state avoids an extra AX call on repeated dictations into the same app.

The recovery theme is just as important as speed. Push-to-talk can occasionally miss a key-up event if macOS event delivery is interrupted or reordered around app focus and permission transitions. When that happens, the app can remain in a fake recording state: the preview overlay stays visible and future recordings are blocked. A small release guard should compare the listener state with the physical key state on macOS, then auto-stop and clear the stale pressed flag when the key is no longer held. Modifier hotkeys should use AppKit's current modifier flags for this guard because raw HID key-state checks can report false negatives for held modifier keys on some systems.

The final speed theme is measurement, not guesswork. Existing latency logs should remain, and any new optimization should be covered by tests that prove the fast path still restores the clipboard asynchronously, still verifies insertion when possible, never lets preview-derived text replace final Qwen output by default, and still reports diagnostic state without forcing routine unverified successes into the main final-text preview.

## Non-Goals

This version should not add cloud LLMs, telemetry, manual Whisper toggles, ASR backend menus, or a model marketplace. It should not auto-install Ollama or auto-run Homebrew commands on behalf of the user. It should not relax v2.1.1 and v2.1.2 safeguards around ASR inference locking, chunked Qwen streaming preview, async clipboard restoration, permission refresh, insertion failure history, or voice undo app boundaries.

## Validation

The minimum validation remains the full pytest suite, compileall, pip check, and git diff whitespace checks. Because this version changes runtime behavior and the menubar UI, it also requires building the DMG, verifying it with hdiutil, installing it over `/Applications/SpeakType.app`, and hand-testing the diagnostics window plus a short dictation with Ollama running and stopped.
