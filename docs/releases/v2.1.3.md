# SpeakType v2.1.3 Release Notes

SpeakType v2.1.3 is a local readiness and latency update for the 2.1 line. The product boundary remains unchanged: audio, raw transcription, polished text, and translation all stay on the user's Mac. Speech recognition remains fixed on Qwen3-ASR through mlx-audio, and text polish/translation continues to use the user's local Ollama service.

## Highlights

- Added a Local Diagnostics window that checks macOS input permissions, microphone discovery, Qwen3-ASR cache/load state, Ollama installation, Ollama service reachability, configured Ollama model availability, and the current focused input target without reading dictated text.
- Added user-facing repair guidance for missing local permissions, stopped Ollama services, missing Ollama models, and input fields that cannot be verified.
- Started local LLM readiness checks and Ollama prewarm in parallel with Qwen3-ASR loading, so cold-start readiness work no longer stacks unnecessarily.
- Added ASR warmup after Qwen model load and Ollama chat-path warmup while recording.
- Reduced insertion latency by trimming pasteboard settle delays, checking Accessibility verification earlier, and reusing one-time focused-process preparation state.
- Changed Electron and browser-like input targets such as Codex, Claude, Chrome, ChatGPT, and Gemini to use paste-first insertion, avoiding repeated false-success Accessibility checks in those editors.
- Kept streaming preview display-only for final insertion quality, while adapting preview cadence for quieter tails and longer recordings.
- Preserved the full final ASR audio buffer by default after testing showed that trimming quiet edges could remove useful acoustic context from short Mandarin phrases.
- Added a missed-keyup recovery guard for push-to-talk recordings so a dropped macOS release event cannot leave the overlay pinned in a fake recording state.

## Known Notes

- The app and DMG are ad-hoc signed, not Developer ID signed or notarized. Gatekeeper rejection from `spctl` is expected under the current release constraint.
- macOS may require users to re-grant Accessibility and Input Monitoring permissions after replacing an existing bundled app.
