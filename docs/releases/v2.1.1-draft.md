# SpeakType v2.1.1 Release Notes

SpeakType v2.1.1 is a latency and reliability update for the local-first 2.1 line. The core product boundary is unchanged: audio, transcript text, ASR, and LLM polish all remain local. ASR stays fixed on Qwen3-ASR, and text polish/translation uses the user's local Ollama server.

## Highlights

- Faster release-to-insert path through in-memory Qwen3-ASR audio input, async ASR/LLM warmup, non-blocking streaming shutdown, async clipboard restoration, and async history persistence.
- More reliable local LLM polish: SpeakType now retries stale Ollama failures, keeps the local model warm, shows a clear one-time fallback notification when Ollama is unavailable, and conservatively strips obvious leading speech fillers before LLM polish when filler removal is enabled.
- Streaming overlay rendering is more robust: display-only sanitization removes ASR special tokens/control characters, and character wrapping improves CJK, paths, and long technical token display.
- Paste-mode insertion now prefers clipboard paste before direct keystroke fallback, which is more reliable for embedded editors such as Codex.
- Paste-mode insertion now verifies focused AX text when available, retries through System Events if raw Quartz paste is ignored, and reports failure instead of logging a false success.
- After a permission reset, SpeakType now watches for macOS input permissions becoming granted while the app is still running and shows a restart prompt so the new grants take effect.
- Missing-permission notifications are shorter, so the system notification body fits in the macOS notification UI.
- Polish auto-language mode now selects a Chinese system prompt for Chinese input and an English system prompt for English input, forbids accidental translation when translation is off, and retries Chinese polish once after obvious Chinese-to-English drift.
- Streaming preview and final Qwen/MLX transcription now share an inference lock, preventing concurrent Metal evaluation on the same model after hotkey release.
- Qwen streaming preview now uses chunked full-buffer decoding instead of native token-delta streaming, matching the final transcription Unicode path and avoiding transient CJK replacement glyphs in the overlay.
- App logs now append across restarts so pre-crash diagnostics survive relaunch.
- Bundled installs are now tracked by app signature fingerprint, so same-version replacement builds reset and re-request macOS input permissions instead of relying only on version number changes.
- The installed 2.1.1 build has been smoke-tested with local Ollama and Qwen3-ASR. The release DMG verifies successfully.

## Fixes

- Text polish no longer appears permanently disabled after Ollama starts late; stale unavailable state is retried.
- Polish+translation can run as a single Ollama request when plugins are disabled.
- Microphone start failures do not leave the overlay stuck in Listening.
- Voice undo refuses to delete across apps and tracks snippet/edit insertions.
- Replace-selection and paste insertion restore clipboard content asynchronously and avoid overwriting user-changed clipboard state.
- WhisperDetector resets quiet/loud counters correctly on silence.
- Delayed overlay reset callbacks no longer corrupt a newly visible overlay session.
- Fast hotkey release during recorder startup no longer produces impossible multi-year recording durations in logs/history.
- Insert failures now show a notification and avoid saving failed insertions to history.
- The bundled py2app `site.pyc` is patched after build so Homebrew `sitecustomize` cannot crash direct bundled-Python smoke tests.
- Qwen/MLX ASR output files are written to temporary paths and removed, preventing `transcript.txt` from being created inside the signed app bundle.

## Known Notes

- The app and DMG are ad-hoc signed, not Developer ID signed or notarized. `spctl` rejection is expected under the current product constraint.
- macOS resets Accessibility/Input Monitoring trust after this bundled update. Users must re-grant those permissions for hotkey-driven dictation.
- In one synthetic TTS end-to-end run, Qwen3-ASR recognized "嗯那个" as "问那个"; the LLM polish chain executed, but it did not infer that ASR error as a filler. This is a correction-dictionary use case rather than a polish availability failure.
