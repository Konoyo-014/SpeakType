# SpeakType v2.1.0 Draft Release Notes

This draft is prepared for review only. Do not publish until final approval.

## Summary

SpeakType v2.1.0 turns the app into a more stable release candidate around the local-first dictation workflow. The release keeps the product contract intact: **audio and transcript processing remain on-device**, **ASR is standardized on Qwen3-ASR**, and low-volume speech is handled by the internal adaptive whisper detector instead of exposing a user-facing backend toggle.

## Highlights

- Unified status overlay for recording, transcribing, polishing, and completion
- Adaptive low-volume detection with automatic local gain boost
- Scene-based polish prompts for email, chat, code, notes, and default contexts
- Correction dictionary, smarter snippets, and history export
- Voice undo for the last dictation
- Real GitHub Releases version check

## Stabilization Work Included Before Release

- Fixed voice undo so it only deletes in the same frontmost app that received the last insertion
- Fixed microphone-open failures leaving the UI stuck in recording state
- Fixed chunked streaming preview crashing on legacy Whisper configs
- Fixed main-thread timer dispatch for recording level updates
- Fixed clipboard restoration in selection replacement flows
- Fixed overlay reset races across rapid hide/show cycles
- Fixed whisper detector silence handling so quiet-state counters do not leak across gaps
- Locked runtime ASR behavior back to Qwen3-ASR even for legacy configs that still contain `asr_backend=whisper`
- Restricted sensitive snippet placeholders like `{clipboard}` and `{env:NAME}` to exact trigger matches only
